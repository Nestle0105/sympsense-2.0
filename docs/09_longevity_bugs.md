## Баг 1 — Символ `↑` рендерится как `1` (критично)

**Симптом:** ЛПНП показывает `13.55` вместо `3.55`, холестерин `15.58` вместо `5.58`, мочевина `19.7` вместо `9.7`, креатинин `1125.3` вместо `125.3`.

**Причина:** поле `value_text` в `lab_results_v1.ndjson` хранит значения с префиксом из PDF-источника: `↑3.55`, `↑5.58`, `↑125.3`. Символ `↑` рендерится как `1` в этом шрифте/браузере.

**Фикс в JS** — в функции `metricDotHtml` добавить очистку перед построением `val`:

```js
const cleanVal = (v) => String(v || '').replace(/^[↑↓▲▼\+]+/, '').trim();
const val = m.value !== null && m.value !== undefined && m.value !== ''
  ? `${e(cleanVal(m.value))}${m.unit ? ' ' + e(m.unit) : ''}` : 'нет данных';
```

Стрелки `↑↓` уже закодированы в статусе маркера (цвет точки) — дублировать их в тексте не нужно.

## Баг 2 — Головные боли / Головокружения: все совпадения склеены в одну строку

**Симптом:** показывает `"Головная боль напряженного типа (), Головная боль напряженного типа (2022)"` — несколько записей через запятую вместо одной последней.

**Причина:** функция `_condition_marker` в `longevity_overview.py` возвращает `latest.get("text")` — одно значение. Где-то в JS или Python все кандидаты объединяются вместо выбора последнего.

**Фикс:**

1. Убедиться, что `_condition_marker` возвращает только одну запись (`_latest_record` уже это делает — не менять логику).

2. В JS функции `metricDotHtml` добавить обрезку длинного `value`:

```js
const cleanVal = (v) => {
  let s = String(v || '').replace(/^[↑↓▲▼\+]+/, '').trim();
  // обрезать по первой скобке если строка длиннее 50 символов
  const cut = s.indexOf('(');
  if (cut > 0 && s.length > 50) s = s.slice(0, cut).trim();
  // жёсткий лимит
  return s.length > 60 ? s.slice(0, 57) + '…' : s;
};
```

## Баг 3 — Timestamp в нечитаемом ISO-формате

**Симптом:** subtitle показывает `"обновлено 2026-06-14T16:03:40.994244+00:00"`.

**Фикс в JS** — добавить функцию форматирования и использовать её в `render()`:

```js
function fmtTs(ts) {
  if (!ts) return '';
  try {
    const d = new Date(ts);
    return d.toLocaleString('ru-RU', {day: 'numeric', month: 'long', hour: '2-digit', minute: '2-digit'});
  } catch(e) { return ts.slice(0, 10); }
}
```

В `render()` заменить:
```js
// было
`обновлено ${payload.generated_at || ''}`
// стало
`обновлено ${fmtTs(payload.generated_at)}`
```

## Баг 4 — "11.5 г" выглядит как граммы

**Симптом:** KPI "данных в системе" показывает `"11.5 г"` — "г" читается как грамм, не год.

**Фикс в JS** — в KPI-блоке горизонта:

```js
// было
`${e(h.data_years ?? 0)} г`
// стало
`${e(h.data_years ?? 0)} лет`
```
